# 比赛导航流程简化说明

## 目的

这个文档记录当前代码链路、确认结论、改动计划和已完成修改。后续如果我继续调整脚本或流程，会同步更新这里。

## 当前代码结论

### 1. 现在不是“导入地图后必须重新编译才能导航”

当前实车导航入口是：

- `start_nav_host.sh`
- 实际启动命令：`ros2 launch pb2025_nav_bringup rm_navigation_reality_launch.py world:=qian1 slam:=False use_robot_state_pub:=False`

代码里 `world` 的作用只是拼接地图路径：

- 栅格图 YAML：`pb2025_nav_bringup/map/reality/<world>.yaml`
- 先验点云 PCD：`pb2025_nav_bringup/pcd/reality/<world>.pcd`

也就是说：

- 建图后得到 `pgm + yaml`，只要放到上述目录并命名一致，就能直接给导航用
- `--symlink-install` 构建方式下，改 launch / yaml / xml 一般不需要重新编译，只需要重启对应节点

### 2. 当前建图链路

`slam:=True` 时会启动：

- `slam_toolbox`
- `point_lio`
- `map_saver_server`

其中：

- PCD 会在退出建图后落到 `point_lio/PCD/scans.pcd`
- 栅格图需要手动执行 `map_saver_cli` 保存出 `yaml + pgm`

### 3. 当前比赛点位链路

当前比赛点位直接硬编码在：

- `ros_ws/src/pb2025_sentry_nav/nav_adam_docker/src/pb2025_sentry_nav/pb2025_sentry_behavior/behavior_trees/rmul/rmul_2026_patrol.xml`

当前至少有这几个关键点：

- 巡逻点 1
- 巡逻点 2
- 补给点

这意味着赛场上如果临时改点，现在要直接改 XML，不够快。

## 我建议的比赛简化流程

### 目标流程

1. 启动建图模式
2. 跑一圈后停建图
3. `map_saver_cli` 导出 `yaml + pgm`
4. 运行一个脚本，把 `yaml + pgm + scans.pcd` 自动放到导航要读的位置
5. 启动“调图模式”导航
6. 在 RViz 里手动点 2D Goal / Nav2 Goal，快速试关键点位准不准
7. 确认无误后，再启动“比赛模式”包含行为树自动跑点

### 为什么这样更省时间

- 省掉“手工复制地图文件、改文件名、找 PCD、怀疑要不要重编”的时间
- 先验证地图和点位，再上行为树，问题定位更清楚
- 调图和正式比赛分成两个模式，不必每次都把整套行为树带起来

## 改动计划

### 计划 1

改造 `start_nav_host.sh`：

- 支持从外部配置文件读取 `WORLD`
- 支持“仅导航调图模式”
- 支持“导航 + 行为树比赛模式”
- 支持控制是否打开 RViz

### 计划 2

增加比赛地图落位脚本：

- 输入 `map_saver_cli` 产出的 `yaml`
- 自动找到同目录 `pgm`
- 自动复制到 `pb2025_nav_bringup/map/reality/<world>.yaml/.pgm`
- 自动复制最新 `point_lio/PCD/scans.pcd` 到 `pb2025_nav_bringup/pcd/reality/<world>.pcd`

### 计划 3

增加桌面一键启动：

- 桌面图标直接调用仓库里的启动脚本
- 默认读取仓库根目录的配置文件
- 以后只需要改配置，不需要改桌面图标

### 计划 4

如果你认可，我下一步还建议继续做：

- 把比赛点位从 XML 抽到单独配置文件
- 提供“一键应用点位配置”脚本

这样赛场临时修点位时就不必手改 XML。

## 已完成修改

### 已完成 1

新增启动配置文件：

- `match_nav_config.env`

用途：

- 配置当前比赛使用的 `WORLD`
- 控制是否启动行为树
- 控制是否打开 RViz

### 已完成 2

新增比赛地图落位脚本：

- `scripts/prepare_match_map.sh`

用途：

- 把导出的 `yaml + pgm` 一次性复制到导航地图目录
- 可同时把 `scans.pcd` 复制到先验点云目录

### 已完成 3

改造启动脚本：

- `start_nav_host.sh`

当前支持：

- 从 `match_nav_config.env` 读取默认配置
- `--world <地图名>`
- `--with-behavior`
- `--without-behavior`
- `--use-rviz`
- `--no-rviz`

用途：

- 调图时只起导航和串口，不起行为树
- 正式比赛时再把行为树一起拉起
- 不再把 `world:=qian1` 这种参数写死在脚本里

### 已完成 4

新增桌面启动图标：

- `start_nav_host.sh`
- `/home/asus/桌面/导航启动.desktop`

设计方式与结果：

- 图标本身不写死地图参数
- 统一读取 `match_nav_config.env`
- 以后只改配置文件，不改桌面图标
- 现在可以直接双击桌面图标启动

## 下一步操作建议

如果你现在就要比赛压流程，建议先按这个顺序用：

1. 建图结束后执行 `map_saver_cli`
2. 执行 `./scripts/prepare_match_map.sh --map-yaml <你的yaml路径> --world qian1`
3. 确认 `match_nav_config.env` 里的 `WORLD="qian1"`、`START_BEHAVIOR="0"`
4. 启动 `./start_nav_host.sh`
5. 在 RViz 手动点几个关键点验证
6. 没问题后把 `START_BEHAVIOR="1"`，再启动一次用于正式比赛
