# 行为树使用指南

## 1. 目标

本文档说明如何查看、切换、修改、扩展当前哨兵行为树策略。

## 2. 当前启用树在哪里

当前目标树在参数文件里指定：

- `ros_ws/src/pb2025_sentry_nav/nav_adam_docker/src/pb2025_sentry_nav/pb2025_sentry_behavior/params/sentry_behavior.yaml`

关键字段：

- `behavior_trees`
- `target_tree`

当前配置是：

- `behavior_trees: pb2025_sentry_behavior/behavior_trees/rmul`
- `target_tree: rmul_2026_bump_supply_center`

## 3. 当前策略树文件

当前树文件：

- `ros_ws/src/pb2025_sentry_nav/nav_adam_docker/src/pb2025_sentry_nav/pb2025_sentry_behavior/behavior_trees/rmul/rmul_2026_bump_supply_center.xml`

如果只想改策略，不想改代码，通常只改这个 XML 就够。

## 4. 启动方式

通常分两部分：

### A. 启动导航

需要先启动 Nav2 及相关导航栈，否则 `SendNav2Goal` 没有 action server 可用。

### B. 启动行为树

命令通常是：

```bash
source ~/ros_ws/install/setup.bash
ros2 launch pb2025_sentry_behavior sentry_behavior.launch.py
```

这个 launch 会同时起：

- `pb2025_sentry_behavior_server`
- `pb2025_sentry_behavior_client`

其中：

- server 负责加载行为树和插件
- client 负责向 server 发送 `target_tree`

## 5. 改策略时优先改哪里

### 只改路点/阈值/逻辑顺序

改 XML：

- `rmul_2026_bump_supply_center.xml`

适合修改：

- 补给点坐标
- 集结点坐标
- 中心战场点坐标
- HP 阈值
- 弹丸阈值
- 等待时间
- 优先级顺序

### 改“当前执行哪棵树”

改参数文件：

- `sentry_behavior.yaml`

只改：

- `target_tree`

### 改“加载哪一组树目录”

也是改：

- `sentry_behavior.yaml`

改：

- `behavior_trees`

## 6. 当前这棵树怎么读

当前树主要由这些子树组成：

- `wait_for_game_start`
- `wait_for_respawn`
- `supply_recovery`
- `recover_after_death`
- `route_to_center_battlefield`
- `rmul_2026_bump_supply_center`

推荐阅读顺序：

1. 先看 `rmul_2026_bump_supply_center`
2. 再看 `supply_recovery`
3. 再看 `recover_after_death`
4. 最后看 `route_to_center_battlefield`

## 7. 现有节点怎么用

当前仓库里策略 XML 常用节点包括：

- `IsGameStatus`
- `IsStatusOK`
- `IsRfidDetected`
- `SendNav2Goal`
- `PubNav2Goal`
- `PublishSpinSpeed`
- `PublishTwist`
- `PublishStance`

常见用途：

- `IsGameStatus`：判断比赛是否开始/是否在某阶段
- `IsStatusOK`：判断血量、热量、弹丸是否达标
- `IsRfidDetected`：判断是否检测到补给区/中心区等
- `SendNav2Goal`：调用 Nav2 action
- `PubNav2Goal`：直接发 `goal_pose`

## 8. 什么时候用 SendNav2Goal，什么时候用 PubNav2Goal

### 用 `SendNav2Goal`

适合：

- 明确需要等待到达结果
- 需要 action 反馈
- 需要“到达后再执行下一步”

### 用 `PubNav2Goal`

适合：

- 持续覆盖式给点
- 不关心 action result
- 想减少 action halt/cancel 带来的问题

当前新策略使用的是 `SendNav2Goal`，因为它更适合“先到点，再进入下一阶段”的路点任务。

## 9. 当前树与 region monitor 的配合方式

当前树不会直接发“颠簸对齐控制”。

当前机制是：

1. 行为树先发导航目标
2. 机器人进入颠簸区
3. `pb2025_region_monitor` 检测到进入多边形
4. `pb2025_region_monitor` 接管底盘模式和速度控制
5. 对齐后直行通过

因此如果要改颠簸区行为，优先看：

- `ros_ws/src/pb2025_sentry_nav/nav_adam_docker/src/pb2025_region_monitor__orig_backup_20260505/pb2025_region_monitor/region_monitor_node.py`

## 10. 如果要新建一棵策略树

推荐步骤：

1. 复制当前 XML 一份
2. 改新的主树 ID
3. 修改内部子树和坐标
4. 在 `sentry_behavior.yaml` 里把 `target_tree` 指过去

例如：

```xml
<BehaviorTree ID="rmul_2026_my_strategy">
```

然后在参数里改：

```yaml
target_tree: rmul_2026_my_strategy
```

## 11. 调试建议

### 看是否真的发了目标

关注：

- `navigate_to_pose`
- `goal_pose`

### 看是否进入补给区

关注：

- `referee/rfid_status`

### 看血量和弹药是否触发阈值

关注：

- `referee/robot_status`

关键字段：

- `current_hp`
- `projectile_allowance_17mm`

### 看 region monitor 是否接管

关注：

- `cmd_vel`
- `cmd_chassis_mode`
- `cmd_yaw_angle`

## 12. 当前最重要的维护点

1. 坐标是否匹配当前地图
2. 补给区 RFID 是否正常
3. region monitor 的颠簸区域多边形是否正确
4. 行为树 target_tree 是否被错误改回旧树
